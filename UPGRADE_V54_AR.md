# تحديث منصة العلوم — الإصدار 54

## ما تم إصلاحه

- كود متابعة الحجز يظهر فور إرسال النموذج في بطاقة واضحة قابلة للنسخ.
- عند إضافة طالب من لوحة المدرس تظهر نافذة بكود الطالب وكود ولي الأمر فورًا.
- عند قبول حجز يتم إنشاء الطالب والأكواد على الخادم في عملية واحدة، ثم يختفي الطلب من قائمة الحجوزات وينتقل الطالب إلى قائمة الطلاب.
- يظل كود متابعة الحجز صالحًا بعد القبول لعرض كود الطالب وكود ولي الأمر.
- الأرقام العربية مثل `٠١٢٣` والفارسية مثل `۰۱۲۳` والإنجليزية تعمل في الهواتف والأكواد.
- يمكن استخدام نفس رقم ولي الأمر لعدة أبناء، ويمكن أن يساوي رقم الطالب.
- الامتحان يقبل أسئلة مكتوبة أو ملف PDF أو الاثنين معًا، بحد أقصى 15MB للـPDF.
- صفحة الامتحانات تفتح مباشرة بالكود المحفوظ لتقليل خطوات دخول الطالب.
- صفحة المدرس لها Manifest مستقل لتثبيت «لوحة المدرس» على الهاتف.
- تم تحديث التخزين المؤقت حتى تعمل النسخة الجديدة من الرابط العادي بدل الحاجة إلى التصفح الخاص.

## ترتيب النشر الإجباري

من داخل مجلد المشروع شغّل:

```powershell
npm install
npm test
npm run firebase:deploy:backend
npm run firebase:hosting
```

أو الأمر الموحد:

```powershell
npm run firebase:deploy:all
```

يجب نشر الـBackend لأن قبول الحجز الجديد يعتمد على Cloud Function باسم `approveBooking`، ثم نشر الاستضافة حتى تصل الواجهة الجديدة وملفات التثبيت إلى الهاتف.

## اختبار سريع بعد النشر

1. افتح الرابط الأساسي في نافذة متصفح عادية وحدّث الصفحة مرة واحدة.
2. أضف موعدًا من لوحة المدرس ثم نفّذ حجزًا تجريبيًا.
3. تأكد أن كود الحجز ظهر فورًا.
4. من لوحة المدرس افتح «الحجوزات» واضغط «قبول وتسجيل الطالب».
5. تأكد أن الكودين ظهرا وأن الطلب اختفى من الحجوزات وظهر الطالب في «الطلاب».
6. ارفع امتحان PDF تجريبيًا وافتحه بكود طالب.
7. من صفحة دخول المدرس اضغط «تثبيت لوحة المدرس على الهاتف».

> إذا بقي جهاز قديم على نسخة سابقة، أغلق الموقع وافتح الرابط الأساسي من جديد. الإصدار 54 يحدّث Service Worker ويزيل Cache الإصدارات القديمة تلقائيًا.
