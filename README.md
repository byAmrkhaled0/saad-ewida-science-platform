# منصة الأستاذ سعد عويضة للعلوم — الإصدار 56.14

منصة تعليمية عربية مبنية بـ HTML وCSS وJavaScript وFirebase، ومجهزة للعمل على الموبايل كتطبيق ويب قابل للتثبيت (PWA). دخول الطلاب وأولياء الأمور يتم بالأكواد، بينما حسابات فريق العمل تستخدم Firebase Authentication.

## تحسينات الإصدار 56.14

- تحميل مكتبة ماسح QR فقط عند الضغط على زر الكاميرا بدل تحميلها مع كل فتح للصفحة.
- تحميل مكتبة Excel فقط عند استيراد ملف Excel، مما يخفض التحميل الأول للوحة الإدارة بأكثر من 1.2MB.
- إيقاف تنزيل ملفات الإدارة وQR وExcel في خلفية الصفحة الرئيسية.
- تشغيل ملفات JavaScript بطريقة `defer` لتحميل Firebase والواجهة بالتوازي من غير تعطيل رسم الصفحة على الموبايل.
- تقديم الملفات الثابتة من كاش التطبيق فورًا في الفتحات التالية، مع تحديثها في الخلفية.
- تنفيذ قراءتي الموعد وإنشاء الكود بالتوازي داخل دالة الحجز لتقليل دورة كاملة مع قاعدة البيانات.
- إعادة محاولة قبول الحجز تلقائيًا مرة واحدة عند أخطاء الشبكة المؤقتة، مع الحفاظ على منع القبول المكرر.

## إصلاحات الإصدار 56.13

- إصلاح عدم ظهور الطالب في التحفيز الشهري بعد تسجيل الحضور والتسميع والواجب بسبب كاش قديم مدته خمس دقائق.
- إضافة رقم تحديث صغير للترتيب يتغير تلقائيًا مع الحضور والتسميع والواجب والحذف والتزامن الكامل، فيُعاد الحساب عند أول فتح مباشرة.
- جعل لوحة الأوائل شهرية فعلًا؛ الحساب يعتمد على سجلات الشهر الحالي بتوقيت القاهرة.
- إضافة زر «تحديث الترتيب» في الصفحة الرئيسية للمراجعة الفورية من الهاتف.
- توسيع الاختبار الآلي ليطابق كل خدمات `MFCloud` المستخدمة مع تنفيذها، وكل Callable في الواجهة مع Function مصدّرة.
- إعادة مراجعة 71 معالج زر مباشر، و42 خدمة Firebase مستخدمة، و21 Callable و23 Function/Trigger.

## إصلاحات الإصدار 56.12

- تشغيل ماسح QR ومكتبة Excel من داخل الموقع بدل `unpkg` و`jsDelivr` لمنع تحذيرات Tracking Prevention وزيادة ثبات الماسح على الموبايل.
- إضافة مكتبات QR وExcel المحلية إلى كاش التطبيق لتظل لوحة المدرس جاهزة بعد تحميلها أول مرة.
- إصلاح تركيز القائمة الجانبية في لوحة الإدارة باستخدام `inert` وإرجاع التركيز للزر الذي فتحها قبل إخفائها.
- تحسين عرض اسم الطالب في لوحة التحفيز الشهري بصورة شخصية مختصرة وبيانات مرتبة وواضحة على الهاتف.
- استبدال عبارة `24/7` غير الواضحة بعبارة عربية مباشرة في إحصائيات الصفحة الرئيسية.

## إصلاحات الإصدار 56.11

- إصلاح أزرار التسميع والواجب بمسار حفظ مباشر سريع لحساب المدرس، مع Cloud Function كمسار احتياطي بدل ظهور خطأ إنترنت عام.
- جعل علامة التسميع أو الواجب تظهر فور الضغط، ثم تتراجع تلقائيًا فقط إذا فشل الحفظ الفعلي، مع منع الضغط المكرر أثناء العملية.
- إصلاح حذف الطالب: نسخة استرجاع سحابية أولًا، أو تنزيل نسخة JSON تلقائيًا ثم حذف الطالب وبواباته وحضوره ودرجاته وواجباته وتسميعه ومحاولاته بأمان.
- مراجعة جميع معالجات أزرار الموقع، ومنع رسائل النجاح قبل تأكيد الحفظ، وإرجاع أي تعديل محلي إذا رفضته قاعدة البيانات.
- إضافة رسائل دقيقة لأخطاء انتهاء الجلسة والصلاحيات والدوال غير المنشورة والاتصال بدل رسالة «تحقق من الإنترنت» لكل الحالات.
- فلترة مواعيد الحجز بالصف المختار فقط؛ لا تظهر مجموعة عامة أو مجموعة تخص صفًا آخر.
- ربط اختيارات الحجز مباشرة بالمواعيد المفعلة في لوحة المدرس، وتعطيل الحجز إذا لم يوجد موعد للصف.
- إزالة صندوق متابعة حالة الحجز والاكتفاء بالكود الموحّد وبوابتي الطالب وولي الأمر.
- تصغير بطاقة الكود والباركود بعد الحجز لتناسب الهاتف، وإصلاح روابط البوابتين وكاش الصفحات ذات `?code=`.
- جعل قبول الحجوزات فوريًا في الواجهة ومنع نافذة النجاح المتكررة، مع استمرار الحفظ الآمن في الخلفية.
- تقليل مجموعات Firebase التي تحملها كل صفحة عامة، وإلغاء الطلبات المكررة للترتيب لتحسين السرعة على الموبايل.
- فتح لوحة الإدارة بالبيانات الأساسية أولًا وتحميل سجلات الحضور والدرجات والمحاولات الثقيلة في الخلفية.
- منع إعادة تحليل وتخزين ملف بيانات الإدارة كاملًا عند كل انتقال أو ضغطة.
- عرض الطلاب على دفعات لتجنب تجميد الموبايل عند وجود عدد كبير من الطلاب.
- جعل طلب الحجز قابلًا لإعادة المحاولة بنفس الأكواد من غير إنشاء حجز أو طالب مكرر.
- إظهار نجاح الحجز فور اكتمال الكتابة، مع إرسال تنبيه المدرس في دالة مستقلة لا تؤخر الطالب.
- إعادة محاولة تسجيل الطالب وفتح بوابة الطالب أو ولي الأمر تلقائيًا عند أخطاء الشبكة المؤقتة.
- إظهار عطل الاتصال في صفحة ولي الأمر بوضوح بدل وصف كل خطأ على أنه كود غير صحيح.
- كود وباركود واحدان للطالب وولي الأمر، مع توحيد أي حساب قديم من لوحة الإدارة.
- إضافة زري التسميع والواجب داخل متابعة الحصة، وحفظ كل علامة في ملف الطالب وتقرير ولي الأمر.
- ربط الترتيب بـ30% حضور و40% درجات و15% واجب و15% تسميع، مع إظهار النسب في الملف والتقرير.
- إصلاح ماسح QR في بوابة الطالب ببديل يعمل على المتصفحات التي لا تدعم `BarcodeDetector`.
- إصلاح طباعة تقرير ولي الأمر وملف PDF الفارغ، وتحويل عرض كل المواعيد إلى نظام 12 ساعة.
- حماية رفع الواجب بتصريح مؤقت من الخادم بدل السماح برفع ملفات غير محدودة بمجرد معرفة كود طالب.
- تنظيف تلقائي لسجلات تحديد المعدل وطلبات الحجز وتصاريح الرفع وجلسات الامتحان المنتهية عبر Firestore TTL.

- تحديث إجباري آمن لملفات الواجهة لمنع بقاء نسخة الحجز القديمة في المتصفح العادي مع عمل النسخة الخاصة.
- إزالة صندوق متابعة حالة الحجز لتبسيط الصفحة وتسريعها على الهاتف.
- سهم رجوع واضح داخل دائرة مستقلة، مع إزالة أزرار التثبيت والمكونات العائمة القديمة.
- استبدال حاوية الرجوع القديمة بزر سهم دائري مستقل يظهر بعد النزول ويعيد المستخدم لأول الصفحة بسلاسة.
- إزالة نهائية لشريط التنقل السفلي القديم ومنع أي تمدد أو تداخل أفقي على شاشات الهاتف.
- أيقونة خدمة مساعدة بشكل سماعة دعم واضحة واحترافية بدل شكل الروبوت القديم.
- قائمة Hamburger جانبية خفيفة على الموبايل في الموقع ولوحة الإدارة بدل ازدحام أزرار الهيدر والتنقل السفلي.
- زر «اسألني» أصبح أيقونة روبوت واضحة ومدمجة على الموبايل، مع الاحتفاظ بكل وظائف مساعد الطالب.
- نقل الطالب من مجموعة لمجموعة، أو إضافة الطالب غير الموزع إلى مجموعة، من قائمة إجراءات الطالب.
- حجز الموبايل يعرض فقط المجموعات المفعلة المطابقة للصف والمضافة من صفحة المواعيد.
- واجهة حجز جديدة للموبايل بخطوات أوضح ومدخلات وأزرار أسهل للمس.
- مركز أدوات مصغّر في صفحة الطلاب يجمع الإضافة، وتقارير الشهر، والاستيراد والتصدير، وترقية الأكواد في أيقونات واضحة.
- تنسيق طباعة موحد بمقاس A4 للتقارير والجداول وملفات الطلاب وأولياء الأمور.
- صفحة رئيسية أكثر حداثة وإلهامًا تعرض رحلة الطالب من الشرح حتى متابعة التقدم.
- صفحة طلاب أقل ازدحامًا: ملخص سريع، صفوف مدمجة، وأهم إجراءين ظاهرين مع تجميع بقية الإجراءات في قائمة واحدة، من غير حذف أي وظيفة.
- تصميم بطاقات واضح للطلاب على الموبايل بدل الجدول العريض.
- أكواد دخول رقمية موحدة من 8 أرقام للطالب وولي الأمر، سواء تم التسجيل من الحجز أو لوحة الإدارة.
- شاشة نجاح الحجز تعرض الكود الموحّد والباركود نفسه مع النسخ وروابط بوابتي الطالب وولي الأمر.
- حماية أقوى لأنواع ملفات الواجبات، وحدود قراءة تمنع تضخم بيانات بوابات الطلاب.
- تحسينات وصول للنماذج والتقييمات والتنبيهات، واختيار الشهر الحالي تلقائيًا.

- ظهور الكود الموحّد فور الحجز أو التسجيل الإداري أو قبول الحجز.
- قبول الأرقام العربية والإنجليزية وتحويلها تلقائيًا، مع السماح بتكرار رقم ولي الأمر ولأن يكون هو نفسه رقم الطالب.
- قبول الحجز من خلال Cloud Function واحدة؛ بعدها ينتقل الطالب لقائمة الطلاب ويختفي الطلب فورًا من قائمة الحجوزات.
- لوحة حجوزات أبسط ومواعيد أسرع، مع وصول مباشر للحجوزات والمواعيد من شريط الهاتف.
- تثبيت لوحة المدرس كتطبيق مستقل على الهاتف من صفحة دخول المدرس.
- إنشاء امتحان بأسئلة مكتوبة أو ملف PDF أو الاثنين معًا، ودخول تلقائي أسهل بالكود المحفوظ.
- تحديث أقوى للـService Worker لتجاوز الملفات القديمة التي كانت تجعل الموقع يعمل في التصفح الخاص فقط على بعض الأجهزة.

- واجهة عامة ولوحة إدارة محسّنتان للموبايل، مع قائمة جانبية Drawer وشريط تنقل سفلي.
- تثبيت المنصة على Android، وتعليمات صحيحة للإضافة إلى الشاشة الرئيسية على iPhone.
- تخزين Firebase موزع على Collections بدل الاعتماد على مستند ضخم واحد.
- ترحيل تلقائي للبيانات القديمة من `settings/siteData` عند أول مزامنة إدارية.
- الحجز والتقييم عبر Cloud Functions فقط مع Rate Limiting، بدون كتابة عامة مباشرة إلى Firestore.
- العام الدراسي والترم في بيانات الطلاب والحضور والامتحانات.
- استيراد الطلاب من CSV أو Excel، وتصدير الطلاب والحضور والدرجات واشتراكات السنتر.
- تسجيل اشتراك السنتر كحالة فقط، بدون دفع إلكتروني.
- نسخ احتياطي سحابي يومي، ونسخ يدوي، واستعادة محمية بكلمة تأكيد.
- صلاحيات أكثر تقييدًا لحساب المساعد.
- مواعيد فتح وإغلاق الامتحان وتحديد الصف والمجموعة والعام الدراسي والترم.
- سجل أخطاء المتصفح داخل لوحة الإدارة.

## التشغيل المحلي

```powershell
npm install
npm run dev
```

ثم افتح:

```text
http://127.0.0.1:5173
```

## الاختبار والبناء

```powershell
npm test
npm run build
```

نسخة النشر تُنشأ داخل `dist/`.

## النشر على Firebase

لنشر القواعد والدوال والاستضافة مرة واحدة:

```powershell
npm run firebase:deploy:all
```

أو على مرحلتين:

```powershell
npm run firebase:deploy:backend
npm run firebase:hosting
```

## متطلبات مهمة

- تثبيت المنصة على الهاتف يتطلب تشغيل الموقع عبر HTTPS.
- النسخ الاحتياطي المجدول يعتمد على Cloud Scheduler، وقد يتطلب خطة Firebase تدعم Cloud Functions المجدولة.
- تفعيل المصادقة الثنائية لحسابات فريق العمل يتم من إعدادات Firebase Authentication / Identity Platform.
- راجع `UPGRADE_V56_AR.md` قبل النشر، خصوصًا ترتيب نشر الدوال ثم الاستضافة.
