# منصة الأستاذ سعد عويضة للعلوم — الإصدار 63.2

## جديد 63.2

- ربط التحفيز الشهري مباشرة بكود الطالب للسنتر والأونلاين بنفس المعادلة دون أي تفرقة.
- إدخال درجة الامتحان التلقائية والتصحيح المقالي في سجل الدرجات والتحفيز فورًا، مع منع تكرار النتيجة في بوابة الطالب.
- تحديث ترتيب التحفيز فور تصحيح الواجب، وإضافة نتيجة الشهر ومركز الطالب وتفاصيل النسب داخل بوابة الطالب.
- قراءة نشاط الشهر الحالي فقط بطريقة مرتبة وقابلة للتوسع بدل اختيار عدد عشوائي من السجلات القديمة.
- اختبارات مستقلة لمعادلة التحفيز، ولتساوي طالب السنتر والأونلاين، ولدخول الطالب في الترتيب حتى قبل وجود درجة امتحان.
- إعادة تصميم أقسام الامتحانات والواجبات ومحاضرات الأونلاين وبنك الأسئلة للموبايل.
- بطاقات محتوى بعناوين وحالات واضحة، وأزرار متساوية لا تتزاحم على الشاشة الصغيرة.
- نوافذ إضافة وتعديل على هيئة لوحة موبايل قابلة للتمرير، مع رأس وزر حفظ ثابتين.
- منشئ امتحان عمودي مريح للموبايل، وترتيب أوضح لنوع السؤال والاختيارات والحذف والتحريك.
- تحسين شاشة تصحيح الامتحانات وتسليمات الواجب وبحث وفلاتر بنك الأسئلة للموبايل.
- رفع رقم الكاش إلى 63.2.0 لضمان ظهور التصميم الجديد بدل النسخة المحفوظة القديمة.

## جديد 63.0

- سجل دفع شهري مستقل للسنتر والأونلاين مع المبلغ والطريقة والتاريخ، وظهوره داخل بوابة الطالب.
- تعديل بيانات الطالب من نافذة كاملة وربطه بموعد مطابق للصف ونوع الدراسة.
- منشئ امتحانات كامل يدعم المسودة والنشر والجدولة والتعديل والنسخ وترتيب الأسئلة والاستيراد من بنك الأسئلة.
- واجبات موجهة حسب النوع والصف والمجموعة، يختارها الطالب عند التسليم ثم يراجعها المدرس ويضيف الدرجة والملاحظة.
- إدارة محاضرات الأونلاين والروابط وملفات PDF مع التعديل والإيقاف وفترة الإتاحة.
- تقارير شهرية موحدة ونسخ احتياطي وإعدادات التواصل داخل لوحة المدرس.
- حماية تسجيل الغياب الجماعي باشتراط تحديد النوع والصف والمجموعة مع تأكيد العدد قبل التنفيذ.
- سعة قصوى للمواعيد تمنع قبول حجوزات فوق العدد المحدد.
- قائمة إدارة مبسطة ومجمعة، وألوان أهدأ ووضع ليلي أوضح وتحسينات شاملة للموبايل.
- رقم إصدار وكاش موحد 63.2.0 في كل الصفحات ونسخة البناء.

## إصلاحات 62.7 المضمنة

- إصلاح تضارب ملف التصميم القديم الذي كان يقسّم قائمة الطلاب إلى 3 أعمدة ويضع عنوان الأعمدة بجوار صف الطالب.
- صف الطالب وعناوينه الآن يستخدمان نفس شبكة الأعمدة ويمتدان بعرض مساحة المحتوى بالكامل.
- انتقال تلقائي لتخطيط من ثلاثة أسطر منظم عند ضيق الشاشة، مع أزرار متساوية وواضحة للموبايل.
- إصلاح فتح بوابة طالب الأونلاين بالكود، مع معاينة فورية وآمنة من زر «تفاصيل» داخل لوحة المدرس.
- ماسح QR موحّد في بوابة الطالب: QR الطالب يفتح الملف، وQR شاشة المدرس يسجّل حضور الأونلاين تلقائيًا.
- منع تكرار تسجيل الحضور عند مسح باركود الحصة أكثر من مرة.
- ترتيب قائمة الطلاب كصفوف أفقية واضحة وسريعة، مع عناوين أعمدة على الكمبيوتر وتخطيط مناسب للموبايل.
- نقل إجمالي عدد الطلاب أعلى الفلاتر بدل ظهوره داخلها.
- إبقاء طلاب السنتر والأونلاين في صفحة واحدة مع تمييز بصري بسيط فقط.

نسخة موحّدة وسريعة من المنصة. لوحة الإدارة تعمل من ملف واحد مبني، ولا تحمل ملفات ترقيات الإدارة القديمة.

- فلاتر حضور مترابطة: نوع الدراسة ثم الصف ثم المجموعات المطابقة فقط.
- باركود حضور مؤقت لطلاب الأونلاين، والحضور يدخل في التقارير والتحفيز الشهري.
- صفحة دفع موحّدة مع فلاتر السنتر والأونلاين، وتظهر الحالة في بوابة الطالب.
- منشئ امتحان يدعم الاختياري، والصح والخطأ، والمقالي مع زر إضافة أسفل آخر سؤال.
- مراجعات وبنك أسئلة يدعمان PDF والصور والأسئلة والتعديل، مع اختيار السنتر أو الأونلاين.
- نشر واجب حسب نوع الدراسة والصف وظهوره في بوابة الطالب.
- إصلاح فتح حساب طالب الأونلاين بالكود حتى للحسابات التي أنشأتها إصدارات أقدم.
- شعار واضح داخل لوحة الإدارة وخلفية وألوان موحّدة ووضع ليلي أفضل.
- تجميع ملفات CSS وJavaScript وقت البناء، وتحميل الصور والمكتبات الثقيلة عند الحاجة.
- رؤوس أمان CSP وتجهيز Firebase App Check للتفعيل التدريجي.
- تحديث حزم Functions؛ فحص الاعتماديات الحالي بلا ثغرات معروفة.

## التشغيل المحلي

على Windows شغّل `start-site.bat`، أو نفّذ:

```powershell
npm install
npm run dev
```

ثم افتح:

```text
http://127.0.0.1:5173
```

صفحة الإدارة:

```text
http://127.0.0.1:5173/teacher-login.html
```

الخادم المحلي يبني `dist/` أولًا ويوقف كاش المتصفح حتى ترى آخر تعديل مباشرة.

## الاختبار والبناء

```powershell
npm test
npm run build
```

نسخة النشر النهائية تُنشأ داخل `dist/`.

## النشر الكامل على Firebase

لنشر قواعد Firestore وStorage فقط بدون نشر الموقع أو الـFunctions، شغّل على Windows:

```text
deploy-firebase-rules.bat
```

سيفتح تسجيل Google الرسمي ثم ينشر `firestore.rules` و`firestore.indexes.json` و`storage.rules` فقط، ولا يحذف بيانات الطلاب.

خصائص كود طالب الأونلاين، الدفع، الواجبات، الامتحانات وباركود الحضور تحتاج نشر الواجهة والدوال والقواعد معًا:

```powershell
npm run firebase:deploy:all
```

أو على مرحلتين:

```powershell
npm run firebase:deploy:backend
npm run firebase:hosting
```

نشر `dist/` فقط يحدّث الشكل، لكنه لا يحدّث منطق Cloud Functions أو قواعد Firebase.

## النشر على Vercel

- Framework: Other
- Build Command: `npm run build`
- Output Directory: `dist`

عند استخدام Vercel للواجهة، انشر Firebase backend أولًا:

```powershell
npm run firebase:deploy:backend
```

## إعداد اختياري للحماية

ضع مفتاح reCAPTCHA الخاص بـ Firebase App Check في `assets/firebase-config.js` داخل `appCheckSiteKey` بعد تسجيل الدومين في Firebase Console. اتركه فارغًا أثناء التجربة المحلية.
