# ترقية المنصة إلى V56.14

هذا الإصدار يحتفظ بكل محتوى المنصة ووظائفها، ويخفف تحميل الصفحة الأولى ولوحة الإدارة على الموبايل، ويسرّع مسار التسجيل والقبول.

## قبل النشر

1. احتفظ بنسخة احتياطية من بيانات المنصة.
2. نفّذ `npm test` ثم `npm run build`.
3. انشر الدوال أولًا ثم ملفات الموقع. قواعد Firestore وStorage والفهارس لم تتغير في هذا الإصدار.

```powershell
firebase deploy --only functions
npm run firebase:hosting
```

أو استخدم الأمر الكامل:

```powershell
npm run firebase:deploy:all
```

## ما تغيّر في صفحة الطلاب

- كل طالب يظهر في صف مدمج على الكمبيوتر وكارت على الهاتف.
- الحالة والحضور ومتوسط الدرجات ظاهرون بسرعة.
- التعديل والملف هما الإجراءان الأساسيان الظاهران.
- الحضور وواتساب ونسخ الكود الموحّد وتغييره والحذف ما زالت موجودة داخل قائمة «إجراءات».
- الحذف وتغيير الأكواد يظلان متاحين للمدرس أو المدير فقط.

## ملاحظات التوافق

- أزرار التسميع والواجب والحضور والدفع والحذف تحتاج نشر قواعد Firestore الجديدة، والحذف السحابي يفضّل نشر Firebase Functions أيضًا.
- إذا لم تكن دالة الحذف السحابي منشورة، سينزّل المتصفح نسخة استرجاع للطالب على الجهاز قبل تنفيذ الحذف المباشر الآمن.
- كل موعد يجب أن يكون مرتبطًا بصف محدد في صفحة المواعيد؛ خيار «كل الصفوف» القديم لن يظهر في الحجز.
- شاشة الحجز تعرض المواعيد المفعلة المطابقة للصف فقط، ولا تسمح بإرسال الطلب من دون موعد صحيح.
- صندوق متابعة حالة الحجز أُزيل، ويستخدم الطالب وولي الأمر الكود الموحّد في بوابتيهما مباشرة.
- قبول الحجوزات يختفي فور الضغط على الهاتف ولا يعرض نافذة نجاح متكررة؛ تظهر رسالة فقط إذا فشل القبول.
- الكود الجديد موحّد للطالب وولي الأمر ويتكون من 8 أرقام، والـQR يحتوي على الكود نفسه.
- لأن النظام بلا طلاب حاليًا، كل تسجيل جديد سيبدأ بالنظام الموحّد مباشرة. زر «توحيد الأكواد» متاح لأي بيانات قديمة تُستعاد لاحقًا.
- الملفات المسموح بها للواجب: JPG وPNG وWebP وPDF، بحد أقصى 10 ميجابايت.
- ترتيب الطالب: 30% حضور، 40% درجات، 15% واجب، 15% تسميع.
- بعد النشر جرّب حجزًا واحدًا، وافتحه من بوابتي الطالب وولي الأمر بنفس الكود، ثم سجّل حضورًا وتسميعًا وواجبًا واطبع تقرير ولي الأمر PDF.
